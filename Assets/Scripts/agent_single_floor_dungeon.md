# agent.md

## 项目目标

使用 Unity 2022.3 LTS 和 C#，在现有 Unity 2D 项目中制作一个可操控的单层俯视角射击地牢 Demo。

当前阶段只验证：

- 玩家移动、瞄准、射击、冲刺手感
- 半颗心扣血制
- 单层房间地图生成与探索
- 普通战斗房清怪流程
- 道具房、商店房
- 炸墙进入的隐藏房
- 类似《以撒》的小地图
- 玩家属性、金币、炸弹和暂停界面

当前阶段不实现错误道具、随机词条、鉴定、日志回放、记事内容、拆解与组装。

不要擅自扩大范围。

---

## 一、基础玩法流程

1. 开始游戏，随机生成一层地牢。
2. 玩家从起始房出发，通过门探索相邻房间。
3. 首次进入普通战斗房后锁门并生成敌人。
4. 清空敌人后开门，房间可能掉落金币、红心、炸弹，也可能没有奖励。
5. 道具房提供一个免费的临时普通强化。
6. 商店房出售三个普通商品。
7. 地图中存在一个不会提前显示的隐藏房。
8. 玩家在正确墙壁旁放置炸弹，炸开入口后可以进入隐藏房。
9. 隐藏房只有在玩家进入后才显示在小地图。
10. 玩家死亡后可以重新开始本局。

本阶段不需要 Boss，不需要下一层入口。

## 二、操作

- `WASD`：八方向移动
- 鼠标：控制瞄准方向
- 鼠标左键：连续射击
- `Space`：向当前移动方向冲刺
- `E`：拾取、购买和交互
- `F`：放置炸弹
- `Esc`：打开或关闭暂停界面

### 冲刺要求

- 有明确冷却时间
- 冲刺期间速度提高
- 冲刺期间不能临时改变方向
- 冲刺期间短暂无敌
- 冲刺结束后恢复正常移动
- 没有移动输入时，默认朝当前瞄准方向冲刺
- 当前阶段不实现擦弹奖励

## 三、玩家生命

生命值使用整数“生命单位”，禁止使用浮点数。

```text
1 颗心 = 2 个生命单位
半颗心 = 1 个生命单位
```

玩家初始状态：

```text
最大生命：6
当前生命：6
UI 显示：3 颗完整红心
```

要求：

- 普通敌人接触伤害暂定为 1 个生命单位
- 治疗半颗心恢复 1 个生命单位
- 治疗整颗心恢复 2 个生命单位
- 当前生命不能超过最大生命
- 生命降到 0 时死亡
- 受伤后有短暂无敌时间
- HUD 正确显示完整心、半心和空心
- 当前阶段只实现红心，不实现魂心、黑心、骨心等血量

建议字段：

```csharp
int CurrentHealthUnits
int MaxHealthUnits
```

## 四、玩家属性

所有属性必须由统一的 `PlayerStats` 管理，禁止移动、射击和投射物脚本分别保存重复数据。

必须包含：

```text
MoveSpeed
FireRate
Damage
Range
ProjectileSpeed
Luck
```

含义：

- `MoveSpeed`：正常移动速度
- `FireRate`：每秒射击次数
- `Damage`：基础投射物伤害
- `Range`：投射物最大飞行距离
- `ProjectileSpeed`：投射物速度
- `Luck`：当前只保存和显示，不参与掉落计算

临时普通强化必须修改 `PlayerStats`，移动和射击系统读取最新属性。

不要让强化直接访问并修改 `PlayerMovement`、`PlayerShooter` 等组件的私有字段。

## 五、战斗系统

### 玩家组件

至少拆分为：

```text
PlayerMovement
PlayerAim
PlayerShooter
PlayerDash
PlayerHealth
PlayerStats
PlayerInventory
PlayerInteractor
```

禁止写成一个超大 `PlayerController`。

### 基础武器

当前只实现一把基础枪：

- 无限弹药
- 按住左键连续射击
- 子弹朝鼠标方向飞行
- 子弹使用当前 `Damage`
- 射击间隔由当前 `FireRate` 决定
- 飞行速度由当前 `ProjectileSpeed` 决定
- 飞行距离由当前 `Range` 决定
- 命中敌人或墙体后销毁

### 敌人

必须实现一种近战敌人：

- 房间开始战斗时激活
- 追踪玩家
- 接触玩家造成半颗心伤害
- 可以被玩家子弹伤害
- 生命归零后死亡
- 死亡时通知所属房间

远程敌人属于可选内容，不得影响本阶段验收。

### 伤害接口

建议使用：

```text
IDamageable
```

玩家、敌人和炸弹爆炸统一通过接口造成伤害，不允许不同系统各写一套扣血逻辑。

## 六、房间地图

地图使用整数网格坐标，每个房间占一个格子。

### 房间类型

```text
Start
Combat
Item
Shop
Secret
```

### 生成要求

- 每局生成 8～12 个普通可见房间
- 所有普通房间必须连通
- 必须有 1 个起始房
- 必须有 1 个道具房
- 必须有 1 个商店房
- 必须有 1 个隐藏房
- 其他房间均为普通战斗房
- 道具房和商店房通过正常房门到达
- 隐藏房不属于正常连通图
- 隐藏房优先放在与至少两个普通房相邻的空网格
- 固定随机种子必须生成相同地图

地图逻辑与房间表现必须分离。

建议数据结构：

```text
DungeonLayout
RoomNode
RoomCoordinate
RoomConnection
RoomType
```

禁止通过扫描场景中的房间对象反向推测地图结构。

## 七、房间运行逻辑

所有房间使用统一尺寸，预留上、下、左、右四个门位及对应墙体。

### 起始房

- 不生成敌人
- 玩家出生在房间中央
- 房门根据地图连接状态开启

### 普通战斗房

- 首次进入时关闭所有有效门
- 生成 1～4 个敌人
- 敌人清空后重新开门
- 已清理房间再次进入时不刷新敌人
- 清房奖励只能是基础资源或无奖励

### 道具房

生成一个免费的普通测试强化，随机选择：

- 最大生命增加 1 颗心
- 伤害提高
- 射速提高
- 移速提高
- 射程提高
- 弹速提高

玩家拾取后立刻生效，房间内不再生成新强化。

这些只用于验证属性系统，不是错误道具。

### 商店房

生成三个商品槽位，可出售：

- 恢复半颗心或一颗心
- 增加最大生命
- 增加伤害
- 增加射速
- 增加移速
- 购买炸弹

要求：

- 商品显示价格
- 金币不足时不能购买
- 购买后扣除金币
- 同一商品只能购买一次
- 购买成功后更新 HUD 和属性

## 八、炸弹与隐藏房

玩家初始拥有 3 枚炸弹。

### 放置炸弹

- 按 `F` 在玩家当前位置附近放置炸弹
- 每次消耗 1 枚炸弹
- 炸弹数量为 0 时不能放置
- 炸弹经过固定延迟后爆炸
- 爆炸对范围内敌人造成伤害
- 爆炸可以伤害玩家
- 爆炸可以触发隐藏墙

建议使用：

```text
IExplosionReceiver
```

### 隐藏墙

- 隐藏房相邻普通房的对应墙体可以被炸开
- 初始不显示裂纹或明显提示
- 爆炸命中后替换为可通行入口
- 不能直接穿墙
- 隐藏入口打开后仍不必立即显示在小地图
- 玩家真正进入隐藏房后才标记为已发现

### 隐藏房奖励

当前只放置基础资源：

- 金币
- 红心
- 炸弹

不生成错误道具。

## 九、小地图

小地图固定在屏幕左上角。

必须显示：

- 当前房间
- 已访问房间
- 已发现但未进入的相邻普通房间
- 起始房标记
- 道具房标记
- 商店房标记

隐藏房规则：

- 未进入前完全不显示
- 炸开入口后可以继续保持隐藏
- 玩家进入后才显示
- 离开后永久保留显示

小地图直接读取 `DungeonLayout` 与房间探索状态。

## 十一、暂停界面

- 设置游戏暂停
- 禁止移动、瞄准射击、冲刺、交互和放置炸弹
- 显示暂停菜单
- 再次按 `Esc` 恢复游戏

暂停菜单包含四个分页：

```text
状态
道具
记事本
日志
```

### 状态分页

```text
移速
射速
伤害
射程
弹速
幸运
```

显示值必须直接读取当前 `PlayerStats`。

### 道具分页

当前只显示：

```text
道具系统暂未实现
```

### 记事本分页

当前只显示：

```text
记事本暂未实现
```

### 日志分页

当前只显示：

日志系统暂未实现

三个占位分页必须可以正常切换，但禁止创建错误道具、笔记、录像或鉴定相关逻辑。

## 十四、验收标准

最终 Demo 必须满足：

- 能从主菜单开始新游戏
- 玩家可以移动、瞄准、连续射击和冲刺
- 玩家初始拥有 3 颗心
- 伤害、治疗和 HUD 正确处理半颗心
- 玩家属性统一由 `PlayerStats` 管理
- 右上角正确显示血量、金币和炸弹
- `Esc` 可以暂停和恢复
- 暂停状态页正确显示六项属性
- 道具、记事本、日志分页可以切换并显示占位内容
- 每局生成一张连通的单层地图
- 地图包含起始房、道具房、商店房和隐藏房
- 普通战斗房进入时锁门，清怪后开门
- 已清理房间不会重复刷怪
- 道具房可以拾取普通强化
- 商店可以使用金币购买商品
- 玩家可以放置炸弹
- 正确墙壁可以被炸开
- 玩家可以进入隐藏房
- 隐藏房进入前不显示在小地图
- 小地图正确显示当前位置与探索状态
- 玩家死亡后可以重新开始
- Unity Console 不存在未处理报错
- Windows 构建可以正常运行
