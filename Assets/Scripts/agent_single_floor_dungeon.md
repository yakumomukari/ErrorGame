# Agent 开发说明：单层俯视角地牢原型

## 1. 当前目标

使用 Unity 2022.3 LTS 与 C# 制作一个可直接运行的俯视角 2D 平面射击原型。

本阶段只验证以下内容：

- WASD 移动
- 鼠标瞄准
- 鼠标左键射击
- Space 冲刺，冲刺期间短暂无敌
- 单层房间制随机地图
- 普通战斗房
- 道具房
- 商店房
- 需要炸墙进入的隐藏房
- 类似《以撒》的小地图
- 以半颗心为最小单位的生命系统，初始 3 颗心
- 玩家属性表：移速、射速、伤害、射程、弹速、幸运
- 游戏 HUD：血量、金币、炸弹、小地图
- Esc 暂停界面与分页骨架
- 普通房清怪、锁门、开门流程

本阶段禁止实现错误道具、词条生成、鉴定、日志、回放、拆解、重组、复杂存档、Boss、多层流程和联机。

## 2. 原型玩法流程

1. 开始新游戏并生成一层地牢。
2. 玩家从起始房出发，通过门进入相邻房间。
3. 进入未清理的普通战斗房后，房门关闭并生成敌人。
4. 清空敌人后房门重新打开，房间可能掉落金币、生命或炸弹，也可以没有奖励。
5. 玩家探索固定存在的道具房和商店房。
6. 道具房提供一个免费的普通测试强化。
7. 商店提供三个可购买的普通测试商品。
8. 地图中存在一个不会显示在小地图上的隐藏房。
9. 玩家在与隐藏房相邻的正确墙面附近放置炸弹，爆炸后打开通道。
10. 玩家进入隐藏房后，小地图才显示该房间。

没有错误道具系统。所有奖励只用于验证基础房间与资源流程。

## 3. 操作

- `WASD`：八方向移动
- 鼠标位置：角色瞄准方向
- 鼠标左键：连续射击
- `Space`：向当前移动方向冲刺
- `E`：拾取、购买、场景交互
- `F`：放置炸弹
- `Esc`：打开或关闭暂停界面

冲刺要求：

- 有冷却时间
- 冲刺期间不能改变方向
- 冲刺期间角色短暂无敌
- 冲刺结束后恢复正常移动
- 不实现擦弹奖励系统

## 4. 玩家生命与属性

### 生命系统

生命值以“半颗心”为最小计算单位，内部禁止使用浮点数。

```text
1 颗心 = 2 个生命单位
半颗心 = 1 个生命单位
```

玩家初始状态：

```text
最大生命：6 个生命单位
当前生命：6 个生命单位
显示结果：3 颗完整红心
```

要求：

- 普通敌人基础接触伤害暂定为 1 个生命单位，即半颗心
- 强力攻击可以造成 2 个或更多生命单位
- 治疗半颗心恢复 1 个生命单位
- 治疗整颗心恢复 2 个生命单位
- 当前生命不得超过最大生命
- 生命降至 0 时玩家死亡
- 受伤后提供短暂无敌时间，防止同一敌人连续碰撞瞬间扣空
- HUD 必须正确显示完整心、半心和空心
- 本阶段只实现红心，不实现魂心、黑心、骨心等扩展血量

建议内部字段：

```text
int CurrentHealthUnits
int MaxHealthUnits
```

### 玩家属性表

所有战斗相关参数必须从统一的 `PlayerStats` 数据源读取，禁止各脚本保存互相独立的重复数值。

本阶段必须包含：

```text
MoveSpeed
FireRate
Damage
Range
ProjectileSpeed
Luck
```

语义要求：

- `MoveSpeed`：玩家正常移动速度
- `FireRate`：每秒射击次数，或由统一转换函数换算成射击间隔
- `Damage`：每枚基础投射物伤害
- `Range`：投射物最大飞行距离；原型中可以通过飞行时间换算
- `ProjectileSpeed`：投射物移动速度
- `Luck`：暂时只保存和显示，不参与错误道具或掉落计算

所有强化必须修改 `PlayerStats`，再由移动、射击和投射物系统读取结果。不要让道具直接修改 `PlayerMovement` 或 `PlayerShooter` 的私有字段。

## 13. 地图结构

地图使用整数网格表示，每个房间占据一个网格坐标。

### 房间类型

```text
Start
Combat
Item
Shop
Secret
```

### 生成要求

- 每局生成 8～12 个可见房间
- 所有可见房间必须连通
- 起始房只能有少量出口，避免地图过散
- 必须生成 1 个道具房
- 必须生成 1 个商店房
- 必须生成 1 个隐藏房
- 其余房间为普通战斗房
- 道具房和商店房必须能够通过正常门到达
- 隐藏房不参与普通连通图
- 隐藏房优先放置在与至少 2 个普通房相邻的空网格
- 隐藏房初始不生成可见门
- 使用固定随机种子时必须生成相同地图

地图数据与房间场景表现必须分离。不要把生成逻辑写在具体房间 Prefab 中。

建议数据结构：

```text
DungeonLayout
RoomNode
RoomCoordinate
RoomType
RoomConnection
```

## 13. 房间实现

所有房间使用统一尺寸，并预留上、下、左、右四个门位和四段可替换墙体。

### 普通战斗房

- 玩家首次进入时关闭所有有效门
- 生成 1～4 个敌人
- 房间清空后打开有效门
- 已清理房间再次进入时不重新生成敌人
- 清房奖励只能是基础掉落或无奖励

### 道具房

放置一个免费的普通测试强化，随机从以下列表选择：

- 最大生命值增加
- 基础伤害增加
- 射速增加
- 移动速度增加

本阶段只需要验证拾取和属性变化，不制作随机词条链。

### 商店房

固定生成三个商品槽位，可以从以下商品中选择：

- 恢复生命
- 增加最大生命值
- 增加基础伤害
- 增加射速
- 购买炸弹

商品显示价格。金币不足时不能购买。购买成功后扣除金币并移除商品。

### 隐藏房

- 初始不显示在小地图
- 普通墙面不能直接穿过
- 与隐藏房相邻的房间对应墙体挂载隐藏墙控制器
- 炸弹爆炸范围命中隐藏墙后，墙体破坏并生成可通行入口
- 进入隐藏房后才将其标记为已发现
- 隐藏房可放置金币、生命、炸弹等基础奖励
- 不显示裂纹等明显提示，调试模式除外

## 13. 战斗系统

### 玩家

玩家至少包含：

```text
PlayerMovement
PlayerAim
PlayerShooter
PlayerDash
PlayerHealth
PlayerInventory
PlayerInteractor
```

不要写成一个巨型 `PlayerController`。

### 武器

本阶段只实现一把基础枪：

- 无限弹药
- 固定射速
- 固定伤害
- 投射物有速度与生存时间
- 投射物命中墙体或敌人后销毁
- 支持后续由属性系统修改伤害和射速

### 敌人

至少实现一种近战敌人：

- 发现玩家后追踪
- 接触玩家造成伤害
- 受到子弹伤害
- 生命归零后死亡
- 死亡时通知当前房间

可选实现一种简单远程敌人，但不能影响核心阶段完成。

### 房门

门只关心以下状态：

```text
Locked
Open
Closed
```

房门不能自行查找敌人。由房间控制器统一管理清房状态并通知门开关。

## 13. 炸弹系统

玩家初始拥有 3 枚炸弹。

放置炸弹后：

- 炸弹延迟爆炸
- 对范围内敌人造成伤害
- 可以伤害玩家
- 可以触发隐藏墙
- 一次放置消耗一枚炸弹
- 炸弹数量为 0 时不能放置

爆炸通过范围检测或伤害接口工作，不允许直接查找隐藏房对象。

建议接口：

```text
IDamageable
IExplosionReceiver
```

## 13. 小地图

小地图固定在屏幕左上角，以网格节点表示房间，避免与右上角生命和资源 HUD 重叠。

必须显示：

- 当前房间
- 已访问房间
- 已发现但未进入的相邻普通房间
- 道具房图标
- 商店房图标
- 起始房图标

隐藏房要求：

- 未进入前完全不显示
- 炸开墙后仍可保持隐藏
- 玩家实际进入后才显示
- 离开后继续保留

小地图直接读取 `DungeonLayout` 与房间探索状态，禁止通过扫描场景对象推测地图。


不要为了原型强行上 ECS，也不要把所有功能塞进 `GameManager`。

## 13. 开发顺序

严格按以下顺序推进，每一步可独立运行后再进入下一步。

### 阶段 A：基础操作

- 玩家移动
- 鼠标瞄准
- 射击
- 冲刺
- 半颗心生命与受伤
- 玩家属性表
- 右上角生命、金币和炸弹 HUD
- Esc 暂停界面与状态分页
- 单个测试房间

### 阶段 B：战斗房

- 敌人
- 敌人死亡
- 房门锁定
- 清房解锁
- 基础掉落

### 阶段 C：地图

- 网格房间数据
- 单层随机生成
- 房间连接
- 房间切换
- 已访问状态

### 阶段 D：小地图与界面整理

- 左上角地图节点
- 当前房间
- 特殊房间图标
- 探索状态
- 暂停界面道具、记事本、日志空白分页

### 阶段 E：特殊房间

- 道具房
- 商店房
- 金币与购买
- 基础固定强化

### 阶段 F：炸弹与隐藏房

- 炸弹资源
- 炸弹爆炸
- 隐藏墙
- 隐藏房揭示
- 小地图更新

## 13. 验收标准

原型完成时必须满足：

- 能从主菜单开始新游戏
- 玩家移动、瞄准、射击和冲刺手感正常
- 玩家初始拥有 3 颗心，伤害和治疗能够正确处理半颗心
- 右上角正确显示血量、金币和炸弹数量
- Esc 能暂停和恢复游戏
- 暂停界面左下栏正确显示移速、射速、伤害、射程、弹速和幸运
- 道具、记事本、日志分页可以切换并显示未实现占位
- 单层地图每局能够随机生成
- 地图中存在道具房、商店房和隐藏房
- 普通房进入后锁门，清怪后开门
- 普通房只掉落基础资源或无奖励
- 道具房可以拾取普通测试强化
- 商店可以使用金币购买商品
- 玩家可以放置炸弹
- 正确墙面被炸后可以进入隐藏房
- 隐藏房进入前不显示在小地图
- 小地图能够正确显示当前房间与探索状态
- 角色死亡后可以重新开始
- Unity Console 无报错
- Windows 构建可以运行

## 13. 明确禁止

当前阶段禁止添加：

- 错误道具
- 随机词条
- 道具鉴定
- 道具日志或录像的实际功能
- 记事本的实际记录功能
- 词条拆解与重组
- Boss
- 多层地图
- 角色选择
- 技能树
- 永久养成
- 联机
- 正式美术和复杂动画
- 大规模对象池优化
- 与当前验收无关的框架预留

先把单层探索与射击手感做稳。不要提前制作后续系统，也不要因为“以后可能需要”而堆抽象层。
